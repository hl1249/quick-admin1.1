# Quick Admin 1.1 — 项目总览

## 项目简介
Quick Admin 是一个基于 **NestJS 11** 的快速开发后台管理系统，采用 **MongoDB** 作为主数据库，支持多租户、动态控制器加载、多存储源（OSS）等特性。  
项目核心目标是——**通过内置的「数据库设计器」自动生成 CRUD 控制器和前端页面模板**，极大提升后台系统的开发效率。

---

## 技术栈

| 层级 | 技术 |
|------|------|
| 运行时 | Node.js (>=22) |
| 框架 | NestJS 11 (Express + Socket.IO) |
| 语言 | TypeScript 5.7 |
| 数据库 | MongoDB + Mongoose 8 |
| 缓存 | cache-manager (内存 / Redis) |
| 认证 | JWT + bcryptjs |
| 鉴权 | 自定义 AuthGuard + PermissionGuard |
| 文件存储 | 本地 / 阿里云 OSS / 腾讯云 COS / 七牛云 |
| 包管理 | pnpm |
| 构建 | NestJS CLI + TypeScript Compiler |

## 核心架构

1. **根模块 (AppModule)**  
   全局注册：AuthGuard → PermissionGuard → ResponseInterceptor → ExceptionsFilter → ValidationPipe  
   引入 DynamicModule（动态加载所有业务控制器）

2. **动态加载机制 (DynamicModule)**  
   通过 `DynamicModule.register()` 自动扫描并注册所有 controller，实现零配置扩展。

3. **数据库 (DbService)**  
   封装 MongoDB 原生操作，支持：查询、插入、更新、删除、聚合、关联查询（foreignDB）、字段选择器（_ / $ DSL）

4. **数据库设计器 (DatabaseDesignController)**  
   可视化创建「Schema」→ 自动生成「CRUD 控制器代码」+「前端页面模板」，大幅减少重复编码。

5. **多 OSS 支持**  
   通过工厂模式对接阿里云、腾讯云、七牛云、本地存储，统一接口。

6. **认证与鉴权**  
   JWT + AuthGuard (Token 验证) + PermissionGuard (接口权限验证)，通过 `@SetMetadata('skipAuth', true)` 跳过认证。

7. **WebSocket 双通道**  
   Admin 管理通道 + Client 客户端通道，支持在线用户池管理。

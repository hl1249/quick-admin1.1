# Skill 1 — 环境配置与启动

## 1. 环境变量 (.env)

项目根目录的 `.env` 文件承载所有运行时配置。  
启动时 `load-env.ts`（在 `main.ts` 第一行 import）会率先加载 `.env`，确保所有模块能正确读取。

### 关键环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `PORT` | 服务端口 | `3000` |
| `GLOBAL_PREFIX` | 全局路由前缀（可为空） | `api` |
| `CORS_ORIGIN` | CORS 允许的源 | `*` |
| `MONGODB_URI` | MongoDB 连接字符串 | `mongodb://127.0.0.1:27017/quick-admin` |
| `JWT_SECRET` | JWT 签名密钥 | 自定义字符串 |
| `JWT_EXPIRE` | JWT 过期时间 | `7d` |
| `CACHE_TYPE` | 缓存类型（memory / redis） | `memory` |
| `REDIS_HOST` | Redis 主机（CACHE_TYPE=redis 时） | `127.0.0.1` |
| `REDIS_PORT` | Redis 端口 | `6379` |
| `REDIS_PASSWORD` | Redis 密码 | 可选 |
| `OSS_TYPE` | 对象存储类型 | `local` / `aliyun` / `qiniu` / `tencent` |
| `LOCAL_UPLOADS_DIR_NAME` | 本地上传目录名 | `uploads` |
| `LOCAL_UPLOADS_ROUTE_PREFIX` | 本地文件路由前缀 | `/uploads` |

## 2. 启动方式

```bash
# 开发模式（热重载）
pnpm start:dev

# 生产模式
pnpm build
pnpm start:prod

# 调试模式
pnpm start:debug

# 测试
pnpm test
pnpm test:e2e
```

## 3. 首次安装

服务启动后，访问：

```
GET /setup/status     → 返回安装状态
POST /setup/install   → 初始化数据库（创建集合、写入默认数据）
```

安装完成后可通过 `/dbList` 查看所有集合。

## 4. 认证说明

- 大部分接口需要 JWT Token（通过 `Authorization: Bearer <token>` 传递）
- 公开接口使用 `@SetMetadata('skipAuth', true)` 跳过认证
- 权限接口使用 `@SetMetadata('permissions', ['xxx'])` 声明所需权限
